axios 的二次封装

拿到接口文档发起请求可能遇到的情况

一切正常：没有出现跨域

出现跨域问题

前端解决跨域

在 vue.config.js 中设置代理，但注意他只开发环境中生效，生产环境是无效的。

解决代理问题后

在各个模式下 路径不对的问题需要设置 项目的 环境变量

  • 开发环境， .env.develoment 他实际是个隐藏文件

    我们创建这么一个文件，然后里面配置一些内容例如：

    VUE_APP_TITLE= 开发环境

    VUE_APP_BASEZ_API="http://fawn.xuexiluxian.cn"

  • 生产环境， .env.production他实际是个隐藏文件

    我们创建这么一个文件，然后里面配置一些内容例如：

    VUE_APP_TITLE= 生产环境

    VUE_APP_BASEZ_API="http://fawn.kaiaki.cn"

这么做就可以在 请求拦截器的时候，判断你现在处于什么模式，对应实际的接口 服务器地址究竟是是谁。

后端解决跨域

前端请求接口进行二次封装：只要你进公司做项目 99 % 封装的（注意：这个封装和有没有跨域压根没有关系）

axios 的二次封装

封装 在request 封装 的意义：

  • url统一管理方便日后更换
  • 在请求拦截器中，可以判断用户是否是登录状态
  • 在响应拦截器中，可以根据返回的status，进行状态选择需要做什么

举个例子：二次封装的 request 文件 一般项目定义在 utils 文件夹中

api 解耦

**其一般放在 api 文件下**

封装在 API 解耦 的意义：

  • 单独维护 api 的请求 可以让项目中的所有的 “请求” 方便管理
  • 很多页面用同一请求，所以不需要写多次，直接封装变量与一个方法请求，后面用到单独传自己对应的参数就行了。

src/api/course.js（对接口进行统一管理，每一个模块（比如这个模块叫course 那就是 course的相关API）的接口都用一个js来独立管理）

在 组件 vue 中可以 实例化进行调用

所以在正常的项目中，一般定义

  • router文件夹 也就是页面哪个是哪个
  • utils 工具类 里面 request 作为二次封装
  • api 文件夹 也就是 api解耦的 各个请求模块对应的 方法集合
